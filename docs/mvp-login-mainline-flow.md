# MVP 登录、进游戏与主线流程

本文档定义首个可玩 Demo 的启动、登录、进入主界面、主线任务和新手引导流程。目标是让玩家在 30 分钟内完成挖宝、鉴宝、打怪、装备、境界突破和离线收益教学。

## 1. 流程总览

```text
启动游戏
  -> 加载本地配置
  -> 检查远程配置版本
  -> 游客登录
  -> 拉取/创建玩家存档
  -> 进入主界面
  -> 执行主线任务和新手引导
```

## 2. 启动流程

| 步骤 | 客户端行为 | 服务端行为 | 失败处理 |
| --- | --- | --- | --- |
| 1 | 显示启动页 | 无 | 本地继续 |
| 2 | 读取本地 `Config/json/manifest.json` | 无 | 缺配置则阻止进入 |
| 3 | 请求 `/config/latest` | 返回最新配置版本 | 失败时使用本地配置 |
| 4 | 请求 `/auth/guest-login` | 创建或识别游客账号 | 失败时提示重试 |
| 5 | 请求 `/game/sync` | 返回存档摘要 | 失败时提示重试 |
| 6 | 进入主界面 | 记录登录事件 | 可继续游戏 |

## 3. 游客登录规则

- 首次启动生成本地 `device_id`。
- 使用 `device_id` 调用游客登录接口。
- 服务端若没有账号则创建 `user_id` 和默认存档。
- 服务端返回 `access_token`、`user_id`、`is_new_player`。
- 客户端保存 token，但关键资产以 `/game/sync` 返回为准。

## 4. 新玩家默认存档

| 字段 | 初始值 |
| --- | --- |
| `realm_id` | `realm_01` |
| `current_map_id` | `map_01` |
| `main_quest_id` | `quest_main_001` |
| `spirit_stone` | 0 |
| `jade` | 0 |
| `appraisal_token` | 0 |
| `equipment` | 新手铲 `weapon_iron_spade_001` |
| `idle.last_claim_at` | 当前服务器时间 |
| `tutorial_step_id` | `tutorial_001` |

## 5. 主界面首屏

首屏不做复杂城镇，直接进入可操作主界面。

| 区域 | 内容 |
| --- | --- |
| 顶部栏 | 头像、境界、战力、灵石、玉 |
| 中央区 | 当前地图、挖宝状态、收益预览 |
| 左侧入口 | 主线任务、日常、图鉴 |
| 右侧入口 | 鉴宝、打怪、装备、背包 |
| 底部导航 | 挖宝、角色、装备、任务、商店 |

## 6. 主线任务顺序

| 顺序 | 任务 | 目标 | 解锁/引导 |
| --- | --- | --- | --- |
| 1 | 第一次挖宝 | 完成 1 次挖宝 | 解锁基础背包 |
| 2 | 获得未知古物 | 获得 1 个古物 | 引导鉴宝入口 |
| 3 | 第一次鉴宝 | 鉴定 1 个古物 | 引导出售或收藏 |
| 4 | 穿戴装备 | 穿戴 1 件装备 | 引导战力变化 |
| 5 | 第一次打怪 | 通关 `stage_001` | 解锁关卡节点 |
| 6 | 挑战精英 | 通关 `stage_005` | 奖励鉴宝符 |
| 7 | 第一次境界突破 | 升到 `realm_02` | 解锁第二张地图目标 |
| 8 | 击败首领 | 通关 `stage_010` | 解锁 `map_02` |
| 9 | 第一次离线收益 | 领取 1 次离线收益 | 完成回流教学 |
| 10 | 开启日常 | 完成 1 个日常任务 | 进入自由循环 |

## 7. 新手引导原则

- 只强制前 5 个关键动作：挖宝、鉴宝、穿装备、打怪、境界突破。
- 每个引导步骤只高亮一个按钮。
- 玩家已经完成目标时，自动跳过对应引导。
- 引导状态服务端保存，避免换设备重复引导。
- 引导不遮挡核心收益反馈。

## 8. 接口需求

| 接口 | 用途 |
| --- | --- |
| `/auth/guest-login` | 游客登录 |
| `/game/sync` | 拉取存档摘要 |
| `/quest/claim` | 领取主线或日常奖励 |
| `/tutorial/complete` | 标记引导步骤完成 |
| `/idle/claim` | 领取挂机/离线收益 |
| `/antique/appraise` | 鉴宝 |
| `/battle/start` | 开始战斗 |
| `/battle/finish` | 结算战斗 |
| `/equipment/equip` | 穿戴装备并刷新战力 |
| `/realm/breakthrough` | 境界突破并扣除材料 |

## 9. MVP 验收标准

- 首次启动能自动游客登录并创建存档。
- 老玩家再次打开能恢复到上次地图、任务和引导进度。
- 玩家能按主线顺序完成前 10 个任务。
- 主线任务领取前，服务端必须校验目标完成状态，客户端只展示 `claimable` 结果。
- 引导步骤能根据任务进度自动推进或跳过。
- 主界面能明确显示当前目标和下一步操作。
- 登录、同步、领取奖励、鉴宝、战斗都支持重复请求幂等。

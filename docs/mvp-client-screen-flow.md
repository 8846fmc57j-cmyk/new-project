# MVP 客户端界面与接口对接文档

本文档定义首个研发版客户端需要做出的页面、状态、按钮、接口调用和错误处理。目标不是做最终美术，而是让 Unity/Cocos 原型能跑通完整核心循环：

```text
启动登录 -> 主界面挂机 -> 挖到古物 -> 鉴宝估价 -> 打怪推进 -> 领取主线 -> 回到挂机
```

MVP 暂不接广告，不做复杂城镇，不做多人社交。所有页面优先服务“看得懂下一步、点得出反馈、能形成循环”。

## 1. 页面结构总览

| 页面 | 研发优先级 | 主要目的 | 首版接口 |
| --- | --- | --- | --- |
| 启动/登录页 | P0 | 自动游客登录、加载配置 | `/config/latest`、`/auth/guest-login`、`/game/sync` |
| 主界面 | P0 | 展示当前地图、挂机收益、主线目标 | `/game/sync`、`/idle/claim` |
| 主线任务面板 | P0 | 给玩家下一步目标和奖励 | `/quest/claim` |
| 新手引导层 | P0 | 高亮关键按钮，推进前 12 步 | `/tutorial/complete` |
| 鉴宝估价页 | P0 | 未知古物鉴定、价值揭晓、出售/收藏 | `/antique/appraise` |
| 打怪关卡页 | P0 | 选择关卡、开始战斗、结算奖励 | `/battle/start`、`/battle/finish` |
| 装备页 | P1 | 查看装备、穿戴、战力变化 | `/game/sync`、`/equipment/equip` |
| 境界页 | P1 | 消耗资源突破境界、提升战力 | `/realm/breakthrough` |
| 背包页 | P1 | 查看道具、材料、古物 | `/game/sync` |

## 2. 启动/登录页

### 2.1 页面元素

| 区域 | 内容 |
| --- | --- |
| Logo 区 | 游戏名、版本号、配置版本 |
| 状态文案 | 加载配置中、登录中、同步存档中、进入游戏 |
| 按钮 | 失败时显示“重试” |

### 2.2 流程

1. 读取本地 `Config/json/manifest.json`。
2. 请求 `/config/latest`，比较配置版本。
3. 若没有本地 `device_id`，生成并保存。
4. 请求 `/auth/guest-login`。
5. 保存 `user_id` 和 `access_token`。
6. 请求 `/game/sync`。
7. 进入主界面。

### 2.3 失败处理

| 失败点 | 处理 |
| --- | --- |
| 本地配置缺失 | 停留在启动页，提示重新安装或联系开发 |
| `/config/latest` 失败 | 使用本地配置继续 |
| `/auth/guest-login` 失败 | 显示重试按钮 |
| `/game/sync` 失败 | 显示重试按钮，不进入主界面 |

## 3. 主界面

主界面是 MVP 的第一屏，不能做成“空城镇”。玩家进入后应立刻看到收益、当前目标和可点击入口。

### 3.1 布局

| 区域 | 显示内容 | 交互 |
| --- | --- | --- |
| 顶部资源栏 | 头像、境界、战力、灵石、玉 | 点击头像进入角色页 |
| 主线目标条 | 当前主线标题、进度、奖励预览 | 点击打开主线任务面板 |
| 地图挂机区 | 当前地图名、挖宝动画、每分钟收益、稀有掉落预览 | 点击“领取收益” |
| 快捷入口 | 鉴宝、打怪、装备、背包 | 打开对应页面 |
| 底部导航 | 挖宝、角色、装备、任务、商店 | MVP 商店只占位，不接广告 |

### 3.2 主界面状态

| 状态 | 视觉反馈 | 触发 |
| --- | --- | --- |
| 正在挖宝 | 循环挖掘动画、收益数字轻微跳动 | 默认状态 |
| 可领取收益 | “领取收益”按钮高亮 | 离线或在线累计达到最小领取量 |
| 有未知古物 | 鉴宝入口出现红点 | 背包存在 `state=unidentified` 的古物 |
| 可挑战关卡 | 打怪入口出现红点 | 战力达到推荐值或主线要求 |
| 主线可领奖 | 主线目标条高亮 | 任务目标完成 |

## 4. 主线任务面板

### 4.1 信息结构

| 字段 | 来源 | 说明 |
| --- | --- | --- |
| 当前任务 ID | `/game/sync.main_quest_id` | 对应 `quests.json` |
| 任务标题 | `quests.name_key` | 首版可先用 key 或临时中文 |
| 任务目标 | `quests.objective` | 例如通关关卡、鉴宝次数 |
| 奖励 | `quests.reward` | 灵石、玉、鉴宝符等 |
| 下一个任务 | `quests.next_quest_id` | 领取后推进 |

### 4.2 按钮状态

| 状态 | 按钮文案 | 行为 |
| --- | --- | --- |
| 未完成 | 前往 | 关闭面板并高亮目标入口 |
| 已完成 | 领取 | 调 `/quest/claim` |
| 已领取 | 下一步 | 刷新任务并继续引导 |

## 5. 新手引导层

新手引导只强制核心动作，不长篇讲解。

| 步骤 | 高亮目标 | 完成条件 |
| --- | --- | --- |
| `tutorial_001` | 主线任务条 | 打开主线面板 |
| `tutorial_002` | 挖宝区 | 完成一次收益领取或获得古物 |
| `tutorial_003` | 鉴宝入口 | 打开鉴宝页 |
| `tutorial_004` | 鉴定按钮 | 完成第一次鉴宝 |
| `tutorial_005` | 装备入口 | 打开装备页 |
| `tutorial_006` | 穿戴按钮 | 穿戴新手装备 |
| `tutorial_007` | 打怪入口 | 打开打怪页 |
| `tutorial_008` | 挑战按钮 | 完成 `stage_001` |
| `tutorial_009` | 境界入口 | 打开角色/境界页 |
| `tutorial_010` | 突破按钮 | 完成第一次突破 |
| `tutorial_011` | 离线收益按钮 | 完成一次收益领取 |
| `tutorial_012` | 日常入口 | 打开日常任务 |

完成引导后调用 `/tutorial/complete`。如果玩家已经完成条件，客户端应自动跳过对应步骤。

## 6. 鉴宝估价页

鉴宝是 MVP 的核心爽点之一，页面要把“未知、揭晓、估值、选择”做清楚。

### 6.1 页面布局

| 区域 | 内容 | 说明 |
| --- | --- | --- |
| 顶部栏 | 鉴宝符数量、今日鉴宝次数 | MVP 可只显示鉴宝符 |
| 左侧列表 | 未鉴定古物、已鉴定古物 | 未鉴定优先排序 |
| 中央展示 | 古物剪影/图标、名称、品级、来源地图 | 未鉴定时隐藏真实价格 |
| 估价结果区 | 基础价、最终估价、稀有度、价格浮动 | 鉴定后显示 |
| 操作区 | 鉴定、出售、收藏、返回 | MVP 先做鉴定和返回，出售可占位 |

### 6.2 鉴定流程

```text
选择未鉴定古物
  -> 点击“鉴定”
  -> 播放揭晓动画
  -> 调用 /antique/appraise
  -> 展示最终估价和结果类型
  -> 标记古物为已鉴定
```

### 6.3 估价反馈

| 结果类型 | 展示方式 | 用途 |
| --- | --- | --- |
| 普通估价 | 白色/绿色价格 | 大多数结果 |
| 小赚 | 蓝色价格、轻微闪光 | 中等正反馈 |
| 大赚 | 紫色价格、弹出短动画 | 稀有正反馈 |
| 极品 | 金色价格、全屏揭晓 | 低概率大奖 |

MVP Mock API 当前返回 `result_type=normal`，后续服务端应根据古物稀有度、随机浮动和玩家加成返回不同结果类型。

### 6.4 空状态

| 情况 | 文案 | 行为 |
| --- | --- | --- |
| 没有未鉴定古物 | 暂无可鉴定古物 | 显示“去挖宝”按钮 |
| 鉴宝符不足 | 鉴宝符不足 | 显示来源说明，暂不跳广告 |
| 接口失败 | 鉴定失败，请重试 | 保留未鉴定状态 |

## 7. 打怪关卡页

打怪是地图推进和装备价值验证的核心。MVP 不需要复杂战斗操作，重点是战力比较、挑战反馈和奖励结算。

### 7.1 页面布局

| 区域 | 内容 |
| --- | --- |
| 顶部栏 | 当前战力、当前地图、已通关最高关 |
| 关卡列表 | 关卡编号、怪物名、推荐战力、首通奖励 |
| 关卡详情 | 怪物头像/图标、血量、攻击、掉落预览 |
| 操作区 | 挑战、扫荡占位、返回 |

### 7.2 关卡状态

| 状态 | 显示 |
| --- | --- |
| 未解锁 | 灰色锁定，显示解锁条件 |
| 可挑战 | 正常显示，挑战按钮可点 |
| 战力偏低 | 挑战按钮可点，但提示“可能失败” |
| 已通关 | 显示首通已领，可重复挑战占位 |

### 7.3 战斗流程

```text
选择关卡
  -> 点击挑战
  -> 调 /battle/start
  -> 播放简化战斗动画
  -> 调 /battle/finish
  -> 展示胜利/失败和奖励
  -> 若主线目标完成，主线条高亮
```

MVP 研发版可以先做 3-5 秒自动战斗：

- 我方和怪物各一个血条。
- 角色自动攻击。
- 怪物自动反击。
- 结果以服务端 `/battle/finish` 为准。

### 7.4 结算弹窗

| 字段 | 说明 |
| --- | --- |
| 结果 | 胜利/失败 |
| 关卡 | `stage_id` |
| 奖励 | 首通奖励、普通掉落 |
| 成长提示 | 战力提升、装备入口、主线可领取 |

## 8. 接口与页面映射

| 页面动作 | 接口 | 成功后 |
| --- | --- | --- |
| 启动检查配置 | `GET /config/latest` | 对比配置版本 |
| 游客登录 | `POST /auth/guest-login` | 保存 token |
| 进入主界面 | `GET /game/sync` | 刷新玩家存档 |
| 领取挂机收益 | `POST /idle/claim` | 更新资源栏 |
| 领取主线奖励 | `POST /quest/claim` | 推进主线 ID |
| 完成引导 | `POST /tutorial/complete` | 推进引导 ID |
| 鉴宝 | `POST /antique/appraise` | 更新古物状态 |
| 开始战斗 | `POST /battle/start` | 创建战斗表现 |
| 结算战斗 | `POST /battle/finish` | 更新关卡和奖励 |
| 穿戴装备 | `POST /equipment/equip` | 更新装备槽位和战力 |
| 境界突破 | `POST /realm/breakthrough` | 更新境界、战力和消耗 |

## 9. MVP 研发验收

- 首次进入游戏不用注册，能自动创建游客账号。
- 玩家第一屏能看到当前目标、收益和核心入口。
- 主线能把玩家引到鉴宝和打怪，不需要玩家猜下一步。
- 鉴宝页能展示未鉴定、鉴定中、已鉴定三种状态。
- 打怪页能展示推荐战力、挑战动画和结算奖励。
- 断网或接口失败时不吞资产、不丢任务状态。
- 所有核心动作都能通过 Mock API 本地跑通。

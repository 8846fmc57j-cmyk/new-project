# MVP 研发任务清单

本文档把当前技术文档拆成可执行研发任务。目标是做出第一个可试玩的研发版，而不是完整商业上线版。

研发版验收目标：

```text
玩家首次启动 -> 自动游客登录 -> 看到主界面 -> 领取收益 -> 鉴宝估价 -> 打怪结算 -> 领取主线奖励
```

MVP 暂不做广告、不做充值、不做复杂社交、不做正式美术资源。先验证核心循环是否成立。

## 1. 里程碑总览

| 里程碑 | 目标 | 产出 | 验收 |
| --- | --- | --- | --- |
| M1 配置与 Mock API | 客户端可读取配置和接口 | JSON 配置、Mock API、本地测试 | `Tools/test_mock_api.py` 通过 |
| M2 客户端启动链路 | 能进入游戏 | 启动页、游客登录、同步存档 | 首次启动进入主界面 |
| M3 主界面挂机 | 能看到收益和下一步 | 主界面、资源栏、主线条、领取收益 | 领取收益后资源增加 |
| M4 鉴宝估价 | 能完成古物鉴定 | 鉴宝页、揭晓动画、估价结果 | 未鉴定古物变为已鉴定 |
| M5 打怪关卡 | 能完成一次战斗 | 关卡页、简化战斗、结算弹窗 | `stage_001` 可通关并发奖励 |
| M6 主线和引导 | 能按目标推进 | 主线面板、引导层、红点提示 | 前 10 个主线任务可连续推进 |

## 2. 客户端任务

### 2.1 基础工程

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 创建客户端工程 | P0 | Unity 或 Cocos 二选一，首版建议 Unity | 能运行空场景 |
| 接入本地配置加载 | P0 | 读取 `Config/json/manifest.json` 和各配置 JSON | 能打印配置版本和表数量 |
| 封装 API Client | P0 | 支持 GET/POST、JSON 序列化、错误回调 | 能请求 `/health` |
| 建立全局状态 Store | P0 | 保存 profile、assets、quest、tutorial、antiques | `/game/sync` 后 UI 可读取状态 |

### 2.2 启动和登录

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 启动页 | P0 | 显示加载状态、版本号、重试按钮 | 网络失败时能重试 |
| 游客登录 | P0 | 生成并保存 `device_id` | 返回 `user_id` 和 token |
| 同步存档 | P0 | 调 `/game/sync`，写入全局状态 | 登录后进入主界面 |

### 2.3 主界面

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 顶部资源栏 | P0 | 显示境界、战力、灵石、玉 | 资源变化后刷新 |
| 主线目标条 | P0 | 显示当前任务、奖励、按钮状态 | 可打开主线面板 |
| 挂机收益区 | P0 | 显示地图、收益预览、领取按钮 | `/idle/claim` 后灵石增加 |
| 快捷入口 | P0 | 鉴宝、打怪、装备、背包 | 可跳转页面 |
| 红点提示 | P1 | 未鉴定古物、可领奖、可挑战 | 状态变化后红点刷新 |

### 2.4 鉴宝估价

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 古物列表 | P0 | 未鉴定优先、已鉴定次之 | 能选中一个古物 |
| 鉴定按钮 | P0 | 未鉴定古物才可点 | 点击后调 `/antique/appraise` |
| 揭晓表现 | P0 | 先用 1 秒遮罩/光效占位 | 结果出现前有等待反馈 |
| 估价结果 | P0 | 显示最终价格、稀有度、结果类型 | 古物状态变为已鉴定 |
| 空状态 | P0 | 无古物时提示去挖宝 | 不出现空白页面 |

### 2.5 打怪关卡

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 关卡列表 | P0 | 显示关卡、怪物、推荐战力 | 可选择 `stage_001` |
| 关卡详情 | P0 | 显示怪物属性和首通奖励 | 数据来自 JSON 配置 |
| 开始战斗 | P0 | 调 `/battle/start` | 返回 battle_id |
| 简化战斗表现 | P0 | 3-5 秒自动战斗，血条变化 | 战斗期间按钮禁用 |
| 战斗结算 | P0 | 调 `/battle/finish` | 胜利后显示奖励 |

### 2.6 主线和新手引导

| 任务 | 优先级 | 说明 | 验收 |
| --- | --- | --- | --- |
| 主线面板 | P0 | 显示任务目标、奖励、前往/领取 | `/quest/claim` 推进任务 |
| 引导遮罩 | P0 | 每次只高亮一个按钮 | 完成后调 `/tutorial/complete` |
| 自动跳过 | P1 | 已完成目标不重复引导 | 老玩家不被卡住 |
| 任务完成检测 | P1 | 根据资产、古物、关卡、引导状态判断 | 主线条能高亮领奖 |

## 6. 每日研发顺序建议

1. 先启动 `python3 Server/mock_api.py`。
2. 客户端先请求 `/health`，确认本地服务可用。
3. 完成启动、登录、同步。
4. 做主界面，只接真实接口，不写假数据到 UI。
5. 做鉴宝页，验证估价反馈。
6. 做打怪页，验证战斗结算。
7. 最后补主线和新手引导，把流程串起来。
